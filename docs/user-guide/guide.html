系统简介与适用范围

了解 DCMS 当前版本可支持的核心业务场景。

01

DCMS 是一个面向 DOTA2 社区赛事的轻量管理系统，当前版本聚焦于赛事组织最常用的流程：玩家库录入、数字 ID 登录、 赛事报名、队长任命、内战选人、拍卖配置与竞价展示。

核心能力 报名、选人、拍卖一体化
页面结构 首页 / 选手拍卖页 / 内战选人页
当前访问方式 公网 IP 临时访问

角色与权限说明

根据不同使用者身份，系统提供差异化的操作入口。

02
角色一

管理员

  • 登录后台
  • 管理玩家库
  • 创建赛事与控制报名状态
  • 任命队长并初始化赛事
  • 创建拍卖、分配预算、暂停或继续拍卖
角色二

普通玩家

  • 使用数字 ID 登录
  • 查看当前赛事
  • 报名或取消报名
  • 浏览公开玩家库
角色三

队长

  • 在内战选人页选择队员
  • 在拍卖页代表队伍出价
  • 查看队伍预算、阵容和当前轮次

首页未登录状态

未登录时可先查看系统概况，并从首页进入不同身份入口。

03

未登录时，首页会展示管理员登录入口、玩家数字 ID 登录入口、当前赛事概览和公开玩家库。访客可以先浏览信息，再决定以哪种身份进入系统。

  1. 管理员输入后台口令后进入后台工作区。
  2. 玩家输入自己的数字 ID 后进入前台首页。
  3. 未登录访客也可以先查看公开赛事信息和玩家资料。
建议在赛事开始前，将管理员使用的后台口令和玩家的数字 ID 预先整理好，便于现场快速登录。
图 1：DCMS 首页未登录状态 包含管理员登录、玩家登录与公开信息展示

玩家登录后的首页

玩家登录后，首页聚焦“赛事报名”和“公开玩家库”。

04

玩家登录后，可直接查看当前可报名赛事、赛事阶段、报名人数和公开玩家库，并可在首页完成报名或取消报名操作。

玩家操作 01

报名赛事

找到开放报名的赛事，点击 报名 即可加入当前赛事。

玩家操作 02

取消报名

如果已经报名，可点击 取消报名 退出当前赛事。

玩家操作 03

查看赛事阶段

首页会显示赛事当前处于报名中、选人中或拍卖中的状态。

玩家操作 04

浏览玩家库

可按关键词、位置和战力排序查看当前公开玩家资料。

图 2：玩家登录后的首页 报名、取消报名与公开玩家库筛选都在同一页面完成

管理员登录后的首页

管理员首页是整套系统的运营控制中心。

05

管理员登录后，首页会切换为后台工作区，集中展示玩家总数、赛事总数、当前报名状态和拍卖状态，并包含玩家库管理、赛事管理和拍卖配置三大区域。

  1. 在玩家库管理中新增、编辑、删除玩家。
  2. 在赛事与报名管理中创建赛事、开启或关闭报名、代玩家报名。
  3. 在赛事报名完成后勾选队长，并点击初始化赛事。
  4. 在拍卖配置管理中为不同队长设置不同预算并创建拍卖。
图 3：管理员登录后的首页 玩家库、赛事与报名管理、拍卖配置管理集中在后台首页完成

玩家库与赛事管理

管理员最常用的两块能力：维护玩家资料与推进赛事报名流程。

06

玩家库管理

  • 新增玩家
  • 编辑玩家资料
  • 删除未进入赛事流程的玩家
  • 按关键词、位置、战力进行筛选

赛事与报名管理

  • 创建赛事
  • 开启或关闭报名
  • 删除赛事
  • 代玩家报名
  • 从报名名单中选择队长并初始化赛事
注意事项：数字 ID 必须唯一且创建后不可修改；队长必须来自当前赛事的报名名单；删除赛事会同时清理关联的内战和拍卖数据。

选手拍卖页

适用于社区赛或拍卖建队场景，支持管理员控制拍卖节奏、队长实时竞价。

07
管理员能力

控制拍卖状态

管理员可启动拍卖，并在需要时暂停或继续倒计时。

队长能力

参与竞价

队长可查看当前拍品、预算和竞价记录，并在允许时点击 我要出价。

系统规则

预算与加价限制

出价必须满足当前加价要求，且不能超过本队剩余预算。

系统规则

自动成交或流拍

倒计时结束后自动成交；若没有有效出价，选手会进入流拍列表。

内战选人页

适用于传统内战建队，强调当前轮次、剩余选手池和队伍战力平衡。

08
页面区域

报名面板

展示当前报名情况，方便所有玩家查看报名列表。

页面区域

队长选人区

当前轮到的队长可从剩余选手池中选择队员并确认。

实时展示各队成员数量和当前总战力，便于控制平衡。

只有当前轮到的队长可以选人，其他队长无法越权操作。

FAQ 01

为什么玩家不能报名？

请检查赛事是否开放报名、玩家是否已报名、登录时是否使用了正确的数字 ID。

FAQ 02

为什么某个玩家不能被设为队长？

队长必须来自当前赛事的报名名单，未报名玩家不能直接被任命为队长。

FAQ 03

为什么队长不能选人或出价？

通常是因为未轮到该队长、赛事尚未初始化、拍卖尚未开始，或预算与加价条件不满足。

本说明书基于当前 DCMS MVP 版本与线上演示环境生成。由于系统数据会随着赛事操作变化，页面中展示的报名人数、队长名单、 拍卖状态等可能与实际实时数据不同。如后续新增 Excel 导入导出、更多角色权限或数据库能力，建议同步更新本说明书。
